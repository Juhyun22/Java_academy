[ 상속(Inheritance) ]
extends = 확장
자식이 부모 파라미터를 받아와야함
상속을 받으면 부모의 기본 생성자가 자동으로 불려지는데,
부모 클래스에 기본 생성자가 없을 경우에는
- 1) 부모 클래스에 기본 생성자를 만든다.
- 2) 자식 클래스의 생성자에서 부모클래스의 생성자를 직접 호출해준다.
필요한 정보만 보여주는 것이 은닉화를 잘하는 구조다
class 중에 필드가 있거나 없는 것도 있다.
 
*[ 다형성(Polymorphism) ]
상속을 기반으로 함
casting 캐스팅 -> 값이 바뀌는걸 알고 써라 / 정확한 값을 알고 써라
캐스팅 생략 가능(자료형 말고..)

up casting 업캐스팅: 자식 -> 부모로 자료형 변환
1. 부모 클래스의 메소드를 사용할 수 있다.
2. 부모 클래스에 있는 메소드를 오버라이드 한 메소드를 사용할 수 있다.
3. 자식 클래스에만 있는 메소드는 사용할 수 없다.
 
down casting: 부모 -> 자식으로 자료형 변환(캐스팅)
- 자식 클래스가 여러개이기 때문에 명확하게 캐스팅 문법을 명시해 주어야 한다.
- 어떤 자식 클래스인지 instanceof 명령어를 사영해서 검사해야한다.
 
 java.lang.ClassCastException -> 어떤 피를 가지고 있는지.. 어떤건지 잘 봐야함
