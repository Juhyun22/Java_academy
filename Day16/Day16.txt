[ 다형성 문제 ]
class는 면접에는 이론이 많이 나옴
다형성이 무엇이냐? 오버로드랑 오버라이드가 무엇이냐?
- 면접에는 이론이 많이 나옴
 
[ 추상 클래스 ]
추상클래스(abstract class): 추상 메소드가 하나라도 존재하는 클래스
추상 메소드(abstract method): 자식클래스가 반드시 구현해야하는 메소드이다.
문법적으로 반드시 구현해야함
추상클래스는 객체로 만들 수가 없다.
new는 자식만 할 수 있다!
 
[ 추상 클래스 문제 ]
환율은 매번 변경되기 때문에 자체적으로 알 수 없어서 외부에서 받아온다.
class = 사용자에게 제공용으로 쓰인다.
 
[ 인터페이스 ]
class가 아님. 인터페이스임. 쩄든 아님
기능을 구현한다. 학원 다니는거랑 비슷. 여러 학원 다닐 수 있다.
자동차 -> 문짝(), 엔진(), 스티어링()의 상세한 내용이 구현 되는거
구현 = 현대차(클래스) 
동물(인터페이스) -> 걷기, 먹기 ... / 고양이 -> 걷기(4족 보행), 먹기(고양이 방식) / 
인간 -> 걷기(2족), ...
 
[ 래퍼 클래스 ]
기본 자료형(primitive class): int, long, float, double,...
참조 자료형(reference type): int[], String
래퍼 클래스는 클래스이므로 기본 자료형에 몇 가지 메소드들을 포함하고 있다.
wrapper class / boxed integer / 
둘 다 객체인 경우에는 주소가 비교되어 .equals()로 비교해야함
integer에는 굳이 new를 써서 쓰지 않음 .. 메모리 낭비
.equals()를 이용해야함
절대 new로 만들지 마세요!
